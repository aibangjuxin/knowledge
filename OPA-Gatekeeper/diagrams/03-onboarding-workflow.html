Team Onboarding Workflow

From New Team Request to Active Tenant — Automated via GitOps

Automation (GitOps)

  • • NamespaceClaim CRD triggers full automation pipeline
  • • GitOps (ArgoCD/Config Sync) auto-applies all resources
  • • 30-day dryrun window before deny enforcement
  • • Team notified at Day 0, Day 23, and Day 30

Template Version Locking

  • • New teams always get latest stable version (v1.2.0)
  • • Template version pinned at onboarding time
  • • Sandbox tenants stay on legacy v1.1.0
  • • Version upgrades handled separately via migration process

Tenant Type Determines Quota

  • • Standard: 8C/32Gi per prod namespace
  • • Privileged: 32C/128Gi + GPU support (needs approval)
  • • Infra: Platform-grade, unlimited within policy
  • • Sandbox: 2C/8Gi, dev-only, no production access
