Multi-Cluster Fleet Architecture

OPA Gatekeeper Policy Management Across Multiple Kubernetes Clusters

Fleet Control Plane

  • • Hub cluster runs Config Sync / Policy Controller
  • • All policy changes originate from Git (single source of truth)
  • • Templates and Constraints versioned and replicated
  • • NAMESPACES: platform, config-management, default

Tenant Isolation

  • • Each team has isolated namespaces per environment
  • • Namespace labels enable policy targeting
  • • No cross-namespace policy bleed
  • • Replica count varies by environment tier

Platform Enforcement

  • • Cluster-wide constraints apply to all namespaces
  • • Security policies: deny by default (cannot be overridden)
  • • Reliability: deny after 30-day dryrun
  • • Tenant-specific: adjustable within platform bounds

OPA Gatekeeper Multi-Cluster • Fleet Policy Controller • GitOps-driven
