Policy Hierarchy & Scope

Three-Layer Enforcement Model: Cluster → Tenant → Namespace

Layer 1 — Platform (Immutable)

  • • Security baseline: deny-only, cannot be overridden
  • • Applies cluster-wide (all namespaces except kube-system)
  • • Examples: K8sAllowedRepos, K8sPSP*, K8sBlockNodePort
  • • Owner: Platform Team only

Layer 2 — Tenant (Adjustable)

  • • Team-configurable within platform upper bounds
  • • Tenant type determines baseline limits
  • • Standard: 8C/32Gi | Privileged: 32C/128Gi | Sandbox: 2C/8Gi
  • • Each tenant gets isolated namespace set

Layer 3 — Namespace (Exemption)

  • • Single-namespace override for time-limited needs
  • • Emergency: 24-72h auto-expiring
  • • Temporary: up to 90 days, requires justification
  • • All exemptions tracked with audit trail
