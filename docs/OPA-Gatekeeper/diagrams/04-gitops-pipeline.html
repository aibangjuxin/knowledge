GitOps CI/CD Pipeline

Policy Change Lifecycle: Author → Review → CI → GitOps Sync → Clusters

CI Quality Gates

  • • Rego lint: opa check + conftest verify
  • • Unit tests: opa test with test cases
  • • Dry-run: kubectl apply --dry-run=server
  • • Policy diff: kubectl gk show diff artifact

Review Requirements

  • • Platform Team: REQUIRED for all changes
  • • Tenant Team Lead: REQUIRED for tenant NS changes
  • • Security Team: REQUIRED for security policies
  • • Protected branch: no force-push, linear history

GitOps Sync

  • • ArgoCD App: automated sync with prune+selfHeal
  • • Server-Side Apply: no drift, field ownership
  • • Per-cluster branches: isolated cluster overrides
  • • Slack notification on every sync completion
