ASM No-Sidecar Gateway 架构

Istio Ingress Gateway TLS 终止 + 无 Sidecar 后端 — 3 段流量

Gateway 命名空间配置

  • • 命名空间标签: istio.io/rev=asm-managed
  • • Pod: image:auto → ASM 自动注入 Sidecar
  • • 注解: inject.istio.io/templates: gateway
  • • TLS: file-mounted cert /etc/istio/gateway-cert/
  • • Gateway CRD: TLS SIMPLE, credentialName 方式
  • • Service: LoadBalancer 443→8443

Runtime 命名空间配置

  • • sidecar.istio.io/inject: "false"
  • • 无 Envoy Sidecar — 明文网络不可达
  • • NetworkPolicy: default-deny + allow-gateway-int:8443
  • • 无 SPIFFE mTLS — 使用 DestRule SIMPLE
  • • 无法使用 AuthorizationPolicy (无 Sidecar)
  • • 这是安全降级方案，仅适合迁移过渡

3 段流量说明

  • 段① Client → GLB → Gateway: TLS 终止
  • 段② Gateway → Runtime: TLS 发起 (SIMPLE)
  • 段③ Pod 内: nginx 自签证书 HTTPS
  • VirtualService 路由 *.team-a.appdev.aibang
  • DestinationRule: tls.mode=SIMPLE → :8443
