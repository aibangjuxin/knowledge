ASM Ingress 流量架构全景

GKE ASM · 外部请求 → ILB → int-istio Gateway → VirtualService → api/svc-runtime → Pod · istiod xDS 控制

流量入口

  • • Client → HTTPS → GCP Internal LB
  • • ILB 通过 PSC 转发到 int-istio Gateway SVC
  • • Gateway SVC selector 匹配 Gateway Pod
  • • Envoy Pod 接收流量, 解密并解析 Host

Gateway CR 配置

  • • selector: istio-ingressgateway 绑定 Pod
  • • servers[].hosts: 监听域名
  • • servers[].port: 端口协议
  • • TLS 证书 (SIMPLE/MUTUAL)

多 Runtime 路由

  • • Gateway Pod 根据 Host 匹配 VirtualService
  • • api-runtime: host → api VirtualService
  • • svc-runtime: host → svc VirtualService
  • • DestinationRule 定义 subsets + mTLS

控制面机制

  • • istiod 下发 xDS 配置到所有 Envoy
  • • VirtualService 定义路由规则
  • • DestinationRule 定义子集/负载均衡/TLS
  • • MutatingWebhook 自动注入 sidecar
