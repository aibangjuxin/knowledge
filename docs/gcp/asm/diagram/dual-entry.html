Gateway 2.0 Architecture - Dual Entry Points

Left: GLB Entry (Original) | Right: Cross-Project PSC Entry with Istio No-Sidecar (NEW)

Entry Point A: GLB Ingress (Original)

  • • Client → GLB → Nginx L7
  • • /teamA/* → Kong Gateway → GKE Runtime
  • • /teamB/* → GKE Gateway → HTTPRoutes
  • • Kong: Auth/Rate/Logging Plugins

Entry Point B: PSC + Istio No-Sidecar (NEW)

  • • PSC Attachment from External Project
  • • IIP → GCP LB → Istio Ingress Gateway
  • • TLS termination at Gateway
  • • TLS origination to Runtime (No Sidecar)

No-Sidecar Key Design

  • • Gateway NS: istio.io/rev=asm-managed
  • • Runtime NS: istio-injection: disabled
  • • Gateway Pod: ASM Managed Sidecar
  • • Runtime Pod: plain nginx, NO sidecar
  • • NetworkPolicy: allow Gateway → Runtime

Port Mapping

  • • LB :443 → Gateway Pod :8443
  • • Gateway :8443 (Envoy listener)
  • • Runtime :8443 (nginx HTTPS)
  • • Gateway readiness: :15021/healthz
