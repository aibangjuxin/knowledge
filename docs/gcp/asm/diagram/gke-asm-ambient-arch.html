GKE ASM Ambient Mode 架构

GKE 默认安装的 ASM (Anthos Service Mesh) Ambient Mode — 无需 Sidecar 的零信任网络

Ambient Mode 核心组件

  • • ztunnel: Node 级 DaemonSet，L4 mTLS 透明加密
  • • Waypoint Proxy: 命名空间级 Envoy 代理
  • • Gateway API: HTTPRoute/GRPCRoute 路由 CRD
  • • 无需 Pod 内注入 sidecar，不改业务容器

流量路径 (Ambient)

  • • Client → ILB → Gateway (int-istio)
  • • Gateway → ztunnel (Node 级 L4 mTLS)
  • • ztunnel → App Pod (原生无 sidecar)
  • • 加密在 Node 层面完成，App 零感知

GKE ASM 默认配置

  • • GKE 1.20+ 自动安装 ASM Ambient Mode
  • • namespace 标签 istio.io/rev=asm-managed
  • • 业务 Pod 无需 istio-injection label
  • • 支持 Sidecar 模式并存 (migration)

GKE ASM Ambient Mode: 零 sidecar 侵入 + Node 级 ztunnel + 命名空间级 Waypoint · 欲了解 Sidecar 模式参见 istio-ingress-flow.html
