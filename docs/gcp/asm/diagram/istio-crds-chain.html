Istio CRD 链路关系

Gateway → VirtualService → DestinationRule → Service → Pod — trace-istio.sh 追踪路径

STEP 1 — Gateway

  • • namespace: int-istio (独立网关NS)
  • • spec.selector 匹配 Envoy Pod 标签
  • • spec.servers[].hosts 匹配请求域名
  • • TLS 模式: SIMPLE / MUTUAL / ISTIO_MUTUAL

STEP 2/3 — VS + DR

  • • VirtualService gateways 引用 Gateway 名称
  • • DR host 匹配 Kubernetes Service 名称
  • • DR subsets 基于 Pod labels (version/v1)
  • • route[].destination.host → FQDN 自动补全

STEP 4/5 — Svc + Pod

  • • Service.spec.selector 匹配 Pod labels
  • • Endpoints 记录 Pod IP : port
  • • Pod 内 istio-proxy 自动注入
  • • Envoy 通过 localhost 劫持 Pod 流量

./trace-istio.sh --url https://api.example.com/v1/health --namespace int-istio
