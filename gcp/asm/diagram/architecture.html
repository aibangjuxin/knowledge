Istio No-Sidecar Architecture - GKE Gateway API

GKE + Istio Minimal Profile + Gateway API + TLS Termination at Gateway + TLS Origination to Backend

Prerequisites

  • • Istiod deployed (minimal profile)
  • • Gateway CRDs enabled
  • • ASM Managed Control Plane
  • • GKE cluster with Istio add-on
istioctl install \
  --set profile=minimal \
  --set components.cni.enabled=false \
  --set values.global.platform=gke \
  --set hub=us-east4-docker.pkg.dev/... \
  --set tag=1.29.2-distroless

Two Traffic Paths

  • /direkt-plain → HTTP :8080
  • • Gateway terminates TLS
  • • Plain HTTP to backend
  • • No encryption between GW and Pod
  • /direkt-https → HTTPS :443
  • • Gateway terminates TLS
  • • Gateway re-encrypts (SIMPLE mode)
  • • TLS to backend with self-signed cert

Key CRDs

  • Gateway - istio-gateway-k8s-net
  • HTTPRoute - Route rules + hostname
  • DestinationRule - TLS SIMPLE mode
  • ReferenceGrant - Cross-namespace
  • Secret - TLS certificates

No-Sidecar Design

  • • Gateway NS: istio.io/rev=asm-managed
  • • Runtime NS: istio-injection: disabled
  • • Gateway Pod: ASM Managed Sidecar
  • • Runtime Pod: plain nginx, NO sidecar
  • • NetworkPolicy: allow Gateway → Runtime
  • • TLS origination at Gateway level
