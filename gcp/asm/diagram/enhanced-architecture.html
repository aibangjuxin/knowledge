Prerequisites

  • Istiod deployed (minimal profile)
  • Gateway CRDs enabled
  • GKE Managed Istiod (NOT ASM/Anthos)
  • GKE cluster with Gateway API support
istioctl install --set profile=minimal
--set components.cni.enabled=false
--set values.global.platform=gke

No-Sidecar Design

  • Gateway NS: istio.io/rev=asm-managed
  • Runtime NS: istio-injection: disabled
  • Gateway Pod: GKE Managed Proxy (gwClass: istio)
  • Runtime Pod: plain nginx, NO sidecar
  • NetworkPolicy: allow Gateway → Runtime
  • TLS via certificateRefs (not mount)

TLS Termination & Origination

  • Termination: Gateway terminates TLS :443
  • Origination: Gateway re-encrypts via DestinationRule
  • Backend: self-signed certs (insecureSkipVerify: true)
  • Two paths per app:
    → /direkt-plain HTTP :8080
    → /direkt-https HTTPS :8443

ReferenceGrant (Cross-NS)

  • Cert Access: Gateway → gateway-secrets ns
  • Service Routing: HTTPRoute → team1-runtime-ns
  • Default deny model: explicit allow required
  • Security: prevents unauthorized cross-ns access
