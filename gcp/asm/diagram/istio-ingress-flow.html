Istio 入口流量架构 — GKE ASM

外部请求 → ILB/PSC → Gateway(int-istio) → VirtualService → Destination → Service → Pod

流量入口

  • • Client 请求 → GCP Internal LB
  • • ILB 通过 PSC 跨 VPC 转发
  • • 到达 int-istio Gateway SVC
  • • Gateway SVC selector 匹配 Deployment
  • • Envoy Pod 接收流量

Gateway CR 配置

  • • servers[].hosts: 监听域名
  • • servers[].port: 端口协议
  • • tls.credentialName: TLS 证书 Secret
  • • selector: app=int-istio-gw 绑定 Pod

控制面机制

  • • istiod 下发 xDS 配置到所有 Envoy
  • • VirtualService 定义路由规则
  • • DestinationRule 定义子集/负载均衡/TLS
  • • Sidecar 截获进出流量

GKE ASM · trace-istio.sh 追踪链: Gateway → VirtualService → DestinationRule → Service → Pod
