原生 ASM 入口流量架构 — GKE Ambient Mode

GKE ASM Ambient Mode (GKE 1.20+ 默认) · Gateway API (HTTPRoute) · ztunnel Node 级 L4 mTLS · 无 Sidecar

Phase 1 · GCP Edge

  • • Client → HTTPS 请求
  • • GCP Internal LoadBalancer (ILB)
  • • 可选: Cloud Armor (WAF 策略)
  • • ILB 通过 PSC 跨 VPC 转发到 IngressGateway

Phase 2 · Gateway API

  • • GatewayClass: istio — 入口路由资源
  • • Gateway CR — 监听 port/host/TLS
  • • HTTPRoute / GRPCRoute — 路由规则 (替代 VirtualService)
  • • IngressGateway Pod — Envoy sidecar 接收流量

Phase 3 · ztunnel Ambient

  • • ztunnel DaemonSet — 每 Node 一个 Pod
  • • L4 mTLS 透明加解密 (Node 级, 无需 sidecar)
  • • 业务 Pod 原生容器, 无 istio-proxy 注入
  • • WaypointProxy (可选) — namespace 级 L7 策略

Phase 4 · ASM Control Plane

  • • istiod (Google 托管) — ASM Managed
  • • xDS 配置下发到 IngressGateway
  • • Ambient Controller → ztunnel + Waypoint
  • • namespace: istio-system · istio.io/rev=asm-managed

Istio 风格入口 (istio-ingress-flow.html)

  • Gateway CR: networking.istio.io/v1beta1
  • 路由: VirtualService → DestinationRule
  • Sidecar: per-Pod Envoy 注入 (istio-proxy)
  • Namespace: int-istio 专属 Gateway
  • mTLS: sidecar 间点对点
  • ASM 角色: 可选插件 (非默认)

原生 ASM Ambient (本图)

  • Gateway CR: Gateway API (networking.x-k8s.io/v1)
  • 路由: HTTPRoute / GRPCRoute (标准 K8s)
  • Sidecar: 无 · 业务 Pod 零侵入
  • Ambient: ztunnel (Node 级 L4 mTLS)
  • WaypointProxy: namespace 级 L7 代理
  • ASM 角色: GKE 1.20+ 默认安装
