Istio 命名空间隔离架构

int-istio 命名空间 (Gateway) 与业务 Runtime 命名空间分离部署模式

int-istio 网关命名空间

  • • 独立命名空间 int-istio 部署 Gateway
  • • Service type: LoadBalancer 暴露给 ILB
  • • 所有业务 VS 通过 gateways 字段引用
  • • 变更 Gateway 不影响业务命名空间

Runtime 业务命名空间

  • • 每个业务 namespace 有独立 VS + DR
  • • Pod 自动注入 istio-proxy sidecar
  • • PeerAuthentication 启用 mTLS STRICT
  • • AuthorizationPolicy 细粒度 L7 控制

跨命名空间路由机制

  • • VS spec.gateways: ["int-istio-gw"] 引用
  • • 跨 NS 引用格式: "namespace/gateway-name"
  • • istiod 统一管理所有命名空间的 xDS
  • • mTLS 自动在所有 namespace 间建立

Gateway 命名空间分离优势: 独立扩缩容、独立变更、独立监控、业务 namespace 零 Gateway 耦合
