Istio Sidecar 注入流程

Pod 创建时 istiod MutatingWebhook 自动注入 Envoy sidecar 流程

注入触发

  • • Namespace 有 istio-injection=enabled label
  • • 或 Pod 有 sidecar.istio.io/inject="true"
  • • ASM GKE: istio.io/rev=asm-managed label
  • • Kube API Server 调用 MutatingWebhook

istio-proxy sidecar

  • • Envoy 代理: L4/L7 流量拦截
  • • 监听 localhost:15090(Prometheus)
  • • 监听 localhost:15000(admin API)
  • • iptables REDIRECT 劫持入站/出站

istio-init init container

  • • Pod 启动时执行一次
  • • 配置 iptables 规则
  • • 流量重定向到 istio-proxy
  • • 启动完成后即终止

./trace-istio.sh 追踪时无需关注注入 — 注入只影响 Pod 内 istio-proxy，CRD 配置链路不变
