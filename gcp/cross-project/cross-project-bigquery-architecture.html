Cross-Project BigQuery · Private Google Access + restricted.googleapis.com Flow

Master GKE Pod → NetworkPolicy → 199.36.153.4/30 (restricted VIP) → tenant BQ API · WI Federation + dataset IAM PGA DEBUG-FOCUSED

Step 1 · Pod 内 (auth-side)

  • • app: Go/Python bq client,默认走 ADC
  • • GKE metadata-server 拦截 token 请求
  • • KSA signed JWT → STS exchange → federated token
  • • impersonate master GSA: master-bq-reader@...
  • • GOOGLE_APPLICATION_CREDENTIALS 必须 unset
  • • KSA annotation = GSA email

Step 2 · egress gate (network-side)

  • • NS NetworkPolicy: 必须 allow 443 → 199.36.153.4/30
  • • Cloud DNS private zone: *.googleapis.com CNAME → restricted
  • • A records: restricted → 199.36.153.4-7
  • • Subnet PGA: --enable-private-ip-google-access
  • • 节点无 external IP = private 路径唯一入口
  • • ⚠ 漏任一项 → i/o timeout / NXDOMAIN

Step 3 · master VPC routing

  • • VPC firewall: allow egress TCP:443 → 199.36.153.4/30
  • • Default route: 0.0.0.0/0 → default-internet-gateway
  • • Google 自动识别 restricted VIP → internal path
  • • restricted VIP: 所有 *.googleapis.com 共享
  • • VPC Flow Logs: 验证 src/dst 都是 internal IP

Step 4 · tenant BQ (resource-side)

  • • bigquery.googleapis.com: Google API frontend
  • • verify GSA token + project scope
  • • dataset IAM: bigquery.dataViewer + jobUser on GSA
  • • 数据返回沿原路径(Pod 拿到 row set)
  • • VPC-SC perimeter(opt): master ↔ tenant 双向 allow
