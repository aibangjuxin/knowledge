4× TLS 终结点

  • • TLS 1 — External GLB 终结，ajbx-public-cert (TrustAsia DV)
  • • TLS 2 — GLB → Backend Service 内部 re-encrypt（仍用同一 cert）
  • • TLS 3 — Producer ILB 终结，ajbx-tenant-vpc-internal-cert (同一 TrustAsia cert, regional)
  • • TLS 4 — ILB → MIG 终结，MIG 自签 IP-SAN cert (SAN: IP:10.0.1.2, IP:10.0.1.3)，见 bs-hc.md
  • • cert SAN: TLS 1/2/3 用 TrustAsia tenant.taobao.caep.uk；TLS 4 用 IP-SAN

1× 内部 TLS hop (升级前是 HTTP)

  • • 位置: ILB → MIG (backend Python https.server :443)
  • • 协议: HTTPS :443，由 MIG 自签 IP-SAN cert 终结
  • • 范围: GCP 私有网络内 (10.0.1.4 → 10.0.1.2/3)
  • • 升级方法: BS --protocol=HTTPS --port-name=https + MIG 装 IP-SAN cert (详见 bs-hc.md)

跨 VPC PSC 桥接

  • • Consumer NEG → Producer SA 通过 GCP 内部 tunnel
  • • 不需要 VPC Peering，不需要 Public IP
  • • Producer 端 IP 对 Consumer 不可见，反之亦然
  • • SA 用 ACCEPT_AUTOMATIC + PSC NAT subnet (10.0.3.0/24)
