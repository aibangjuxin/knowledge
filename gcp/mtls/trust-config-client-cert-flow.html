Trust Config Client Cert 轮换 — 验证流程

Old Cert vs New Cert 走同一 Trust Config 链验证 — Root + Intermediate 不变,新旧 cert 等价通过

Client Cert 变什么

  • • Subject (CN=user01) — 不变
  • • Issuer (Intermediate CA) — 不变
  • • AKI (指向 Intermediate) — 不变
  • • Serial (0x1A2B3C → 0x4D5E6F) — 变了,不影响
  • • 公钥 (A1B2 → C3D4) — 变了,不影响
  • • Validity (旧时间段 → 新时间段) — 变了,只要新未过期

Trust Config 验证逻辑

  • • Step 1: 读 client cert 的 Issuer DN
  • • Step 2: 在 Trust Store 找匹配的 Intermediate CA
  • • Step 3: 验证 AKI = Intermediate 的 SKI
  • • Step 4: 验 Intermediate 的 Issuer = Trust Anchor
  • • Step 5: 验 Trust Anchor 在 trustAnchors 里
  • • 全部通过 → ✅ 信任建立

Trust Config 不变 vs 必须变

  • • 用户换 client cert → Trust Config 不变
  • • 切 Root CA → 必须变 (旧 cert 全失效)
  • • 轮换 Intermediate → 必须变 (建议双签过渡)
  • • 吊销 CA → 必须变 (移除)
  • • GCP 隐式 chain depth 限制 ~10 层 (gcp/mtls/SSLVerifyDepth.md)
