Cloud SQL Auth Proxy → PSC → Cloud SQL :3307 Debug Flow

GKE Pod (loopback 5432) → NS NetworkPolicy → PSCEP :3307 → Cloud SQL PSC Service Attachment DEBUG-FOCUSED

Step 1 · Pod 内

  • • app: source code,
  • • 用 `127.0.0.1:5432` 连本地
  • • cloud-sql-proxy listen 在 5432 (你的 --port)
  • • ss 看得到 5432,看不到 3307
  • • 因为 3307 不是 Pod 内 listen 端口

Step 2 · NetworkPolicy (关键关卡)

  • • 管控出 Pod 的流量
  • • 默认 deny-all = 全拦
  • • 必须显式放 3307 才能出去
  • • 同时建议放 5432 / 6432 / 443 兼容
  • • ⚠ 配错端口 = i/o timeout

Step 3 · PSCEP (Consumer VPC)

  • • PSC Endpoint IP (192.168.64.104)
  • • 在 Consumer VPC 里的内网 IP
  • • 把包转发到 GCP 托管的 PSC tunnel
  • • proxy 调 Admin API 才知道这个 IP
  • • 服务端固定 listen 3307

Step 4 · Producer (Cloud SQL)

  • • PSC Service Attachment (server side)
  • • 服务端固定 listen 3307
  • • 校验 OAuth token → IAM API
  • • 查 GRANT 决定 DB-level 权限
  • • token 申请是 client 干的,这里是 "验"
