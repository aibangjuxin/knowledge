aibang. 内部域名推荐架构

VPC2 GKE → Peering Zone → VPC1 Forwarding Zone → 企业 DNS

推荐架构优势

  • • VPC2 不需要直接承载企业 DNS target
  • • VPC1 集中管理 forwarding、日志、策略
  • • Cloud DNS Peering 是原生能力
  • • 不强依赖 VPC Network Peering

直接建 Forwarding Zone 条件

  • • 企业 DNS target 对 VPC2 可达
  • • 防火墙允许 35.199.192.0/19 → target:53
  • • 不依赖第三个 VPC 传递路由
  • • 多 target + ILB HA 已设计好

Zone 分层建议

  • • aibang. → 根域（承载少量核心记录）
  • • appdev.aibang. → 平台环境入口
  • • team*.appdev.aibang. → 团队边界
  • • api*.team*.appdev.aibang. → 具体服务
