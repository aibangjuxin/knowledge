ndots 与 Search Path 行为详解

GKE Pod /etc/resolv.conf 中 ndots=5 时，相对域名如何被放大查询

ndots 规则

  • • dots < ndots → 先追加 search path，再查原始名
  • • dots ≥ ndots → 先查原始名字，再追加 search
  • • 尾部点 FQDN（.）→ 直接绝对查询，跳过 search
  • • ndots 越大，企业域名越容易触发 search

Search Path 放大后果

  • • 产生 6~7 次无用查询
  • • 日志里出现 cluster.local 等噪音
  • • DNS 延迟增加
  • • 可能触发企业 DNS 异常

推荐方案

  • • 企业/跨 VPC 域名用尾部点 FQDN
  • • Java OkHttp 封装 AbsoluteFqdnDns
  • • 保持 ndots:5 不变（K8s 服务发现兼容）
  • • 局部低 ndots 只在明确场景用
