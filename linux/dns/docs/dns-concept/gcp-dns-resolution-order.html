Cloud DNS Name Resolution Order

GKE Pod → Cloud DNS 入口 → VPC 解析顺序（Longest Suffix Match）

Longest Suffix Match 原则

  • • Cloud DNS 按最具体后缀命中 zone
  • • 不是 Peering 永远优先于 Forwarding
  • • 例子: appdev.aibang. 优先于 aibang.
  • • 私有/转发/对等 zone 都在同一阶段

Forwarding Zone 关键点

  • • Target DNS 需允许 35.199.192.0/19
  • • 可指定 FQDN target（不只 IP）
  • • 缓存时间 = min(TTL, 60s)
  • • 建议搭配 ILB + 多后端 HA

Peering Zone vs Forwarding

  • • Peering 是单向关系，不要求 VPC Peering
  • • 切到 Producer VPC 后重新走完整顺序
  • • 适合中心化 DNS 管理场景
  • • 推荐架构: VPC2 Peering → VPC1 Forwarding
