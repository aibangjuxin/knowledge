GKE DNS 三种模式

Pod /etc/resolv.conf nameserver 决定 DNS 查询路径

169.254.20.10 — NodeLocal DNSCache

  • • Autopilot 默认启用
  • • Standard 1.34.1+ 默认启用
  • • 同节点缓存，降低延迟
  • • 避免 conntrack UDP 竞态
  • • 未命中则转发上游

169.254.169.254 — Cloud DNS for GKE

  • • Standard Cloud DNS provider
  • • Pod 直连 GCP 元数据 server
  • • Cloud DNS 私有 zone / Peering
  • • 服务记录自动同步

10.x.y.10 — kube-dns Service

  • • Standard 默认 DNS provider
  • • ClusterIP 为 kube-dns
  • • CoreDNS / kubedns Pod 支撑
  • • 可配合 NodeLocal 使用
