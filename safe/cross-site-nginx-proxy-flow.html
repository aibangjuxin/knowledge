Cross-Site Cookie Solution

Option 2 — Recommended

Single Nginx Proxy Architecture — Browser sees same origin for all cookies

How Option 2 Solves the Problem

  • • All traffic routes through single Nginx proxy at www.aibang.com
  • • Browser sees one origin for both frontend and API calls
  • • Cookies set by API backend appear to come from same origin
  • • No dependency on fragile domain similarity heuristics
  • • Works with internal/private API domains

Cookie Attributes (Required)

  • • SameSite=Lax — allows same-site + safe cross-site GET
  • • Secure — required for SameSite=None; always recommended
  • • HttpOnly — prevents XSS script access
  • • Domain=aibang.com — share across subdomains
  • • SameSite=None requires Secure — Firefox blocks without it

Static Content Caching

  • • Nginx proxy handles all traffic — static load concern
  • • Mitigation: enable proxy caching for static assets
  • • Cache TTL: 1 day for CSS/JS/images/fonts
  • • API calls: proxy_buffering off (no caching)
  • • X-Cache-Status header for debugging
