Cross-Nginx Header Rewrite Architecture

External Domain (www.abjx.com) → Nginx → Internal Kong DP (www.aibang.com)

Step 1 — Client Request

  • • 请求域名: www.abjx.com
  • • 路径: /apiname1/endpoint
  • • 协议: HTTPS (TLS)
  • • Host 头部: www.abjx.com
  • • TLS 证书: *.abjx.com 公网 CA

Step 2 — Nginx (Header Rewrite)

  • • server_name: www.abjx.com
  • • SSL termination (公网证书)
  • • proxy_set_header Host www.aibang.com
  • • X-Original-Host: $host (保留原始)
  • • upstream: Kong DP :8000 (HTTP)

Step 3 — Kong DP (Internal)

  • • 仅监听: www.aibang.com
  • • preserve-host: "false"
  • • Route: /apiname1/* → Runtime-1
  • • Route: /apiname2/* → Runtime-2
  • • X-Original-Host 用于日志追踪

Step 4 — GKE Runtime

  • • 收到请求: Host = www.aibang.com
  • • 路径: /apiname1/endpoint
  • • 可选: X-Original-Host = www.abjx.com
  • • 无需感知外部域名
  • • 内部服务发现 (ClusterDNS)
