Cross-Site Cookie — Same-Domain Multi-Path Routing

Single Origin (www.abjx.com) → Nginx → Kong DP (GKE) / External GCP Project | First-Party Cookie

Cookie Flow

  • • Browser → www.abjx.com (single origin)
  • • All Set-Cookie: Domain=www.abjx.com
  • • Cookie Domain matches URL origin
  • • Browser classifies as First-Party ✅
  • • Firefox/Safari: Not blocked ✅
  • • No dependency on domain similarity

Nginx Path Routing

  • • /apiname1/* → Kong DP (HTTP)
  • • /apiname2/* → Kong DP (HTTP)
  • • /ext-crm/* → HTTPS + SNI (other GCP)
  • • /ext-erp/* → HTTPS + SNI (other GCP)
  • • Kong routes: Host rewrite + preserve-host:false
  • • X-Original-Host preserved for tracing

Required Cookie Attributes

  • • SameSite=Lax — recommended
  • • Secure — always required
  • • HttpOnly — prevents XSS theft
  • • Domain=www.abjx.com — explicit
  • • Path=/ — global share
  • • CSRF Token on POST/PUT/DELETE

Cross-GCP HTTPS

  • • proxy_ssl_server_name on (SNI)
  • • proxy_ssl_verify on (CA validation)
  • • proxy_ssl_trusted_certificate (CA bundle)
  • • mTLS if required (client cert)
  • • X-Original-URI preserved for logging
  • • Extended timeouts (10s connect)
