# v7 URL Map 说明

## 目标

本文件说明 [v7.json](/Users/lex/git/knowledge/ssl/docs/claude/routeaction/maps-format-and-verify/success/v7.json) 中新增和修正后的 URL Rewrite 逻辑。

本次配置的目标是：

- 外部浏览器继续访问长域名
- GLB 在内部把请求转发到短域名 `ppd01-ajbx.short.fqdn.aibang`
- 同时把请求路径改写到目标后端需要的前缀

## 生效逻辑

### 1. `lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang`

匹配 Host：

- `lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang`

改写规则：

- `Host` 改写为 `ppd01-ajbx.short.fqdn.aibang`
- `/` 改写为 `/lex-long-fqdn`
- `/*` 改写为 `/lex-long-fqdn/*`

示例：

- `https://lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang`
  -> 后端收到 `https://ppd01-ajbx.short.fqdn.aibang/lex-long-fqdn`
- `https://lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang/abc`
  -> 后端收到 `https://ppd01-ajbx.short.fqdn.aibang/lex-long-fqdn/abc`

### 2. `gg-env-region.projectID.env.aliyun.cloud.region.aibang`

匹配 Host：

- `gg-env-region.projectID.env.aliyun.cloud.region.aibang`

改写规则优先级：

1. 精确匹配 `/jccl/v1/callbackup`
2. 其他请求走 `/` 和 `/*` 默认规则

精确规则：

- `Host` 改写为 `ppd01-ajbx.short.fqdn.aibang`
- `/jccl/v1/callbackup` 改写为 `/v1-sit`

默认规则：

- `Host` 改写为 `ppd01-ajbx.short.fqdn.aibang`
- `/` 改写为 `/gg-env-region`
- `/*` 改写为 `/gg-env-region/*`

示例：

- `https://gg-env-region.projectID.env.aliyun.cloud.region.aibang`
  -> 后端收到 `https://ppd01-ajbx.short.fqdn.aibang/gg-env-region`
- `https://gg-env-region.projectID.env.aliyun.cloud.region.aibang/test`
  -> 后端收到 `https://ppd01-ajbx.short.fqdn.aibang/gg-env-region/test`
- `https://gg-env-region.projectID.env.aliyun.cloud.region.aibang/jccl/v1/callbackup`
  -> 后端收到 `https://ppd01-ajbx.short.fqdn.aibang/v1-sit`

## 本次修正点

- 修复了 `lex-long-fqdn` 路径前缀中的拼写错误：`/ex-long-fqdn` -> `/lex-long-fqdn`
- 为两个长域名补充了 `/*` 规则，避免只有根路径 `/` 能命中改写
- 保留 `gg-env-region` 的特殊回调路径优先转发到 `/v1-sit`

## 验证建议

建议验证以下请求：

- `curl -k -H 'Host: lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang' https://<lb-ip>/`
- `curl -k -H 'Host: lex-long-fqdn.projectID.env.aliyun.cloud.region.aibang' https://<lb-ip>/abc`
- `curl -k -H 'Host: gg-env-region.projectID.env.aliyun.cloud.region.aibang' https://<lb-ip>/`
- `curl -k -H 'Host: gg-env-region.projectID.env.aliyun.cloud.region.aibang' https://<lb-ip>/test`
- `curl -k -H 'Host: gg-env-region.projectID.env.aliyun.cloud.region.aibang' https://<lb-ip>/jccl/v1/callbackup`

验证重点：

- 后端收到的 `Host` 是否为 `ppd01-ajbx.short.fqdn.aibang`
- 后端收到的 URI 是否为预期改写后的路径
- 特殊路径 `/jccl/v1/callbackup` 是否优先命中 `/v1-sit`

我已经确认了关键限制：GCP URL map 支持通配 Host，也支持路径模板重写，但变量只能从 Path 捕获，不能从 Host 捕获后回写到 Path。这意味着“*.long-domain 自动改写成 /$subdomain”这种需求，单靠 GLB URL map 不能优雅完成，最稳的简化方式是把“泛域名匹配”放在 GLB，把“从 Host 派生目标 Path”的逻辑下沉到 Nginx/Kong。
